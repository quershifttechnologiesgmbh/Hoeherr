v1 Baseline - Reference point. SoccerTrack only, frozen backbone. Achieved test mAP@50=0.423 (best overall due to matched train/test domain) but poor ball detection (0.059). 50 epochs, ~2.3h training time.
v2 Combined Data - Same architecture as v1 but trained on merged SoccerTrack + TeamTrack soccer_side data. Adding mixed-viewpoint data hurt overall mAP (0.309 vs 0.423) due to domain mismatch between top-down drone and side-view cameras. Best val mAP@50=0.309 at epoch 48. Precision=0.561, Recall=0.307. Ball detection slightly improved (~0.07 vs 0.059).
v3 Unfrozen Backbone - Fully trainable backbone (freeze=0) on combined data. Early stopped at epoch 46 (best at epoch 38, patience=10). Test mAP@50=0.383, a major improvement over v2 frozen (0.309). Ball detection improved to 0.091 (best of all experiments). Unfreezing was critical for adapting COCO features to mixed viewpoints.
v4 YOLOv8x - Extra-large model (~68M params vs 25.9M) with unfrozen backbone. Best overall test mAP@50=0.391, best player detection (0.692). Best val mAP@50=0.388 at epoch 50 (still improving). Training time ~5.8h. Ball AP50=0.090, nearly matching v3.

Experiment Comparison

Key Findings

  • Mixed-viewpoint data hurts with frozen backbone: v2 (combined + frozen) scored 0.309 vs v1's 0.423 on SoccerTrack test set due to domain gap
  • Unfreezing the backbone is critical: v3 (combined + unfrozen) recovered to 0.383, a +24% improvement over v2
  • YOLOv8x (v4) achieves best overall results: Test mAP@50=0.391, Player AP50=0.692
  • v3 has marginally better ball detection: 0.091 vs 0.090 for v4
  • v1 remains best on matched domain: Its 0.423 test mAP reflects the advantage of training on the same distribution as the test set
  • SAHI dramatically improves ball detection: +565% relative improvement (0.012 to 0.083) on a 200-image subset

SAHI (Sliced Aided Hyper Inference) Evaluated

SAHI addresses tiny object detection by slicing high-resolution images into overlapping patches, running detection on each patch, then merging results with NMS post-processing. Evaluated on a 200-image subset using the v1 baseline model.

How SAHI Works

Input: 3840x2160 frame (or 6500x1000 panoramic) Step 1: Slice into 640x640 patches with 20% overlap (128px) Step 2: Run YOLOv8 on each patch (ball appears ~40-50px in patch = detectable) Step 3: Map detections back to original coordinates Step 4: NMS to remove duplicates at patch boundaries Output: Combined detections

SAHI trades off player detection accuracy for dramatically improved ball detection. A hybrid approach (standard for players, SAHI for ball) may yield optimal results.
